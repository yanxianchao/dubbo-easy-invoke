# Dubbo Easy Invoke

在 IntelliJ IDEA 里直接完成 Dubbo 接口调试，不再在控制台、文档和 IDE 之间来回切换。

Dubbo Easy Invoke 提供从“接口发现”到“调用验证”再到“结果分析与收藏复用”的完整链路，目标是把日常联调变成稳定、可重复、可追踪的操作。

## 核心能力
- 注册中心接入：Settings 配置 Zookeeper 地址，支持缓存优先加载、后台刷新、网络异常回退
- 接口发现与搜索：应用/接口联动下拉，支持关键字实时模糊匹配
- 快速调用：参数与结果自动 JSON 格式化，支持 `粘贴入参`、`复制入参`、`复制结果`
- 接口解析：展示服务地址、消费应用、超时时间、二方包版本、DUBBO 版本
- 收藏复用：收藏当前接口与入参，支持按应用筛选、关键字搜索、重命名、删除、一键回填
- 调试 Console：加载/调用/解析全过程分色输出，状态与关键信息清晰可追踪

## 适用场景
- 日常接口联调与问题复现
- 多应用、多接口场景下的快速定位
- 高频调用参数模板沉淀与复用
- 服务地址、消费者、超时与版本信息核对

## 快速开始

### 环境要求
- JDK 21
- IntelliJ IDEA 2025.2+

### 本地运行插件
```bash
./gradlew runIde
```

### 打包
```bash
./gradlew buildPlugin
```

插件 ZIP 默认输出到：`build/distributions/`

## 使用方式
1. 打开 `Settings > Tools > Dubbo Easy Invoke`，配置 Zookeeper 地址（如 `127.0.0.1:2181`）。
2. 在 IDEA 右侧打开 `Dubbo Invoke` 面板，选择应用与接口。
3. 输入入参后点击 `调用接口`，结果会自动格式化展示。
4. 点击 `解析接口`，查看服务地址、消费应用、超时时间、二方包版本和 DUBBO 版本。
5. 点击 `收藏当前` 保存接口+入参，或在 `收藏夹` 中搜索后回填继续调用。

## 入参说明
- 支持直接输入 Dubbo 表达式：`1, "abc"`
- 支持 JSON（对象 / 数组 / 标量），会自动格式化
- JSON 数组会按原结构透传，例如输入 `[123]` 会按 `[123]` 调用，不会被改写

## 缓存与数据新鲜度
- 首次加载优先命中缓存（内存或磁盘），显著降低打开等待时间
- 命中缓存后自动后台刷新，保证可用性与数据新鲜度平衡
- 点击 `刷新` 可强制拉取最新注册中心数据
- 注册中心暂不可用时会自动回退到本地缓存
