# 收藏接口与调用参数功能设计方案（草案）

## 目标
在现有 Dubbo 调用面板中增加“收藏接口 + 收藏入参”能力：
- 收藏保存到本地磁盘
- 可搜索、可选择收藏项
- 选择后自动带出接口与入参（不自动调用）

## 推荐界面方案
1. 在“接口”这一行右侧新增按钮：`收藏当前`
2. 在“接口”下面新增一行：`收藏夹`
3. `收藏夹`这一行左侧使用可输入搜索的下拉框，右侧放两个按钮：
   - `应用收藏`
   - `删除收藏`
4. 选择收藏后，不自动调用，仅执行：
   - 自动选中接口
   - 自动回填入参

## 交互细节
1. 点击 `收藏当前` 时弹出小对话框
2. 对话框字段：
   - `收藏名`（必填）
   - `收藏夹`（可选，默认“默认”）
   - `保存当前入参`（默认勾选）
3. 保存成功后，收藏夹下拉立即可搜索
4. 在收藏夹下拉输入关键字，实时过滤（匹配收藏名、应用名、接口名）
5. 点击某条收藏并点 `应用收藏` 后：
   - 切换到对应应用
   - 定位对应接口
   - 入参框回填收藏时保存的参数（按现有 JSON 美化规则处理）
6. `删除收藏` 删除当前选中的收藏项，并做一次确认

## 展示文案建议
- 收藏下拉每项展示：`收藏名 | service.method | 应用名`
- 占位提示：`输入关键字搜索收藏接口`

## 本地存储建议（JSON）
保存到本地磁盘一份 JSON 文件，字段建议：
- `id`
- `name`
- `folder`
- `app`
- `service`
- `method`
- `host`
- `port`
- `params`
- `createdAt`
- `updatedAt`

## 待确认规则
1. 同名收藏：覆盖还是不允许重名
2. `应用收藏` 时若当前 ZK 找不到该接口：提示并停止，还是继续回填参数
3. 是否需要“编辑收藏名/收藏夹”
4. 收藏夹是否只做一级分类（如：默认、订单、用户）
